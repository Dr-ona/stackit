# Stackit

[![Flutter CI](https://github.com/Dr-ona/stackit/actions/workflows/flutter-ci.yml/badge.svg)](https://github.com/Dr-ona/stackit/actions/workflows/flutter-ci.yml)

> Highlight anywhere → understand the exact meaning → remember it permanently.

Stackit is an offline-first multilingual vocabulary collector for Android. Select
text in any app, tap **Understand with Stackit**, and get instant translations
with spaced-repetition review — all without leaving your current app.

## OpenAI Build Week

Stackit is entered in the **Education** category. It addresses a common language-
learning gap: useful words appear while people are reading, messaging, or browsing,
but saving them with enough context to learn later is disruptive. Stackit turns a
system text selection into a contextual vocabulary item, then schedules adaptive
practice for each meaning.

### How Codex and GPT-5.6 were used

Codex with GPT-5.6 was used during the final Build Week iteration to audit the
implementation and test surface, align the project with the judging requirements,
improve the setup and judging documentation, and prepare a reproducible submission.
The key product decisions remain visible in the code: an Android-native capture
bridge keeps collection fast, the offline dictionary provides a private baseline,
AI is a fallback rather than a hard dependency, and FSRS schedules review per sense
instead of treating every translation as the same card.

See [the Devpost submission draft](docs/devpost-submission.md) for the judging
walkthrough, demo outline, and final submission checklist.

## Features

### Capture
- Android `ACTION_PROCESS_TEXT` and Share intent integration
- **Surrounding sentence capture** — preserves the original sentence context when the source app allows it
- **App name, URL, and timestamp** — every capture remembers where it came from
- **Privacy gate** — source app, URL, and context data are stored locally by default and only synced to the cloud when you explicitly opt in
- **Clipboard capture** — paste any text from clipboard into Stackit
- **Word vs phrase detection** — badges distinguish single words from multiword expressions
- Manual word/phrase entry with full detail expansion
- Automatic source language detection (English, Arabic, French)
- Per-capture translation route override with wheel picker
- All 9 language routes: EN↔AR, EN↔FR, AR↔FR, and same-language study
- Duplicate detection before saving with "View existing" option

### Meaning
- 87,000+ entry offline dictionary (EN↔AR, EN↔FR, FR↔EN) with AR↔FR pivot through English
- **Gemini fallback** — when a word isn't found offline, AI provides translations and caches them for future offline use
- **Pre-cache** — common words are automatically pre-cached in the background on app start
- Multi-sense vocabulary model with translations, definitions, and examples
- **Find all meanings** — Gemini AI expands entries into distinct senses
- **Contextual explanation** — AI explains a word in its original sentence context
- **Bilingual examples** — auto-generated for senses that lack examples
- **Context-aware ranking** — senses ranked by relevance to your original sentence
- **IPA, transliteration, gender, inflections** — generated by AI per sense
- **Register/domain labels** — formal, informal, technical, regional, and more
- **Semantic relations** — synonyms, antonyms, collocations, idioms
- **Provenance badge** — see whether a meaning came from the offline dictionary, AI, or manual entry
- **Report incorrect meaning** — flag wrong or outdated senses directly from the detail view
- **Sense selection** — choose which senses to save before committing

### Review
- **FSRS adaptive scheduling** — intervals adjust per sense based on how well you remember it
- **Exercise types** — cloze, multiple-choice, reverse translation, definition matching
- **Exercise modes** — new words, weak meanings, overdue review, or mixed
- **Weak-meaning detection** — identifies senses with low stability or high error rates
- **Due reason explanations** — see why each card is due (new, overdue, weak, maintenance)
- **Streak & retention tracking** — consecutive review days and estimated retention percentage
- **Accessible/text-only mode** — full exercise support without audio or speech input
- Daily review reminders via local notifications

### Library
- **Collections and tags** — organize vocabulary into user-created collections with tags
- **Quick add to collection** — one-tap from capture preview, word detail, or library tile
- **Favorites** — star any entry for quick access
- **Smart groups** — language, source, new, due, weak, mastered
- **Search** — full-text search across words, translations, definitions, examples, IPA, synonyms
- **Filters** — by language, status, source, due state, mastery level, with sort orders
- **Multi-select** — select multiple entries for bulk collection, favorite, or delete actions
- **Duplicate merge** — find and merge duplicate captures without losing review history
- **Bulk actions** — multi-select add/remove collections, tags, favorites, delete
- **Entry editing** — edit senses, definitions, examples, part of speech inline
- **Safe collection deletion** — remove a collection without deleting vocabulary
- **Import/export** — JSON, CSV, Anki-compatible formats

### Profile & Sync
- Firebase Authentication (email/password + Google Sign-In)
- Cloud sync with Firestore (bidirectional merge, conflict-safe)
- **Sync status** — last synced time, entry count, and "Sync now" button in Account settings
- **Undo delete** — swipe-to-delete and bulk delete show undo SnackBars
- Learning profile: languages, proficiency, goals, interests
- **Pronunciation/accent preference** — choose US, British, Australian, or Indian English; Saudi, Egyptian, Moroccan, or Emirati Arabic; France, Canada, or Belgian French
- Profile avatar upload and management
- JSON vocabulary export

### Languages
- Full interface in English, Arabic (RTL), and French
- Follows device language or manual override

## Getting Started

### Prerequisites
- Flutter SDK 3.12+
- Android Studio or VS Code with Android tooling
- A Firebase project with the `google-services.json` placed in `android/app/`

### Run

```sh
flutter pub get
flutter run
```

### Build release APK

```sh
flutter build apk --release
```

The signed APK will be at `build/app/outputs/flutter-apk/app-release.apk`.

## Rebuilding the offline dictionary

After downloading and extracting FreeDict 0.6.3 to the documented third-party
path, rebuild the compact asset with:

```sh
dart run tool/build_offline_dictionary.dart
```

## Try the capture flow

1. Run the app on an Android device.
2. Open any app that supports text selection (browser, news, e-reader).
3. Highlight a word or phrase.
4. Choose **Understand with Stackit** from the action menu.
5. Read the meaning and tap **Save for review**.

## Review schedule

Stackit uses **FSRS** (Free Spaced Repetition Scheduler), an adaptive model
that adjusts review intervals per sense based on your actual recall history.
New senses start in a learning phase, then graduate to longer intervals as
you demonstrate recall. Senses you struggle with are shown more frequently.

Exercise sessions come in four modes:
- **New words** — recently collected senses not yet reviewed
- **Weak meanings** — senses with low stability or high error rates
- **Overdue review** — senses whose scheduled review date has passed
- **Mixed** — balanced mix of all the above
